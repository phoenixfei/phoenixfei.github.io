在数组与字符串问题中，有一类题如果还在用 O(n²) 暴力枚举子数组，那说明：该用滑动窗口了。
滑动窗口的本质是：
> **在保持某种条件的前提下，用两个指针维护一个“合法区间”。**

---

## 基本概念

假设我们要统计某种满足条件的子数组：

暴力思路：

```
枚举左端点 l
    枚举右端点 r
        判断区间是否合法
```

时间复杂度：O(n²)

滑动窗口思路：

```
右端点不断右移
左端点按需右移
始终维护一个合法区间
```

时间复杂度：O(n)

**每个元素最多进窗口一次，出窗口一次。**

---

## 通用模板

```python
left = 0
for right in range(n):
    # 扩展窗口
    加入 nums[right]

    # 若不合法
    while 不满足条件:
        移除 nums[left]
        left += 1

    # 此时窗口合法
    更新答案
```

## 两大类型

---

### 固定长度窗口

窗口大小固定为 k：

```python
for i in range(n):
    加入 nums[i]
    if i >= k:
        移除 nums[i-k]
```

典型题：

* 长度为 k 的最大和
* 滑动窗口最大值
* 固定长度子串统计

特点：

* 不需要 while
* 左指针跟随右指针移动

---

### 可变长度窗口（最重要）

窗口大小动态变化：

```python
while 条件不满足:
    left += 1
```

典型题：

* 最长无重复子串
* 最小覆盖子串
* 和 ≥ target 的最短子数组

特点：

* 右扩张
* 左收缩
* 始终保持“合法”

---

## 三种常见题型模型

---

### 模型一：最长型问题

例如：

* 最长无重复子串

核心：

```
窗口合法 → 更新最大长度
窗口非法 → 收缩
```

代码模板：

```python
left = 0
ans = 0
for right in range(n):
    加入 nums[right]
    while 不合法:
        移除 nums[left]
        left += 1
    ans = max(ans, right - left + 1)
```

---

### 模型二：最短型问题

例如：

* 和 ≥ target 的最短子数组

核心：

```
窗口合法 → 尝试收缩
```

代码模板：

```python
left = 0
ans = inf
for right in range(n):
    加入 nums[right]
    while 满足条件:
        ans = min(ans, right - left + 1)
        移除 nums[left]
        left += 1
```

---

### 模型三：计数型问题

例如：

* 至多 K 个不同字符的子串数量
* 恰好 K 个不同字符

经典技巧：

```
恰好 K = 至多 K - 至多 (K-1)
```

非常重要。

## 其他知识点

👉 **滑动窗口常见辅助结构**

| 结构   | 作用     |
| ---- | ------ |
| 哈希表  | 统计频率   |
| 计数数组 | 字符串问题  |
| 双端队列 | 维护单调性  |
| 前缀和  | 快速求区间和 |

---

👉 **滑动窗口 vs 双指针**

区别在于：

| 类型   | 是否维护“合法区间” |
| ---- | ---------- |
| 双指针  | 不一定        |
| 滑动窗口 | 一定         |

> 滑动窗口是“双指针的特殊情况”。

---

👉 **常见错误**

* ❌ 忘记 while
* ❌ 收缩时没更新状态
* ❌ 把“最长型”写成“最短型”
* ❌ 计数型忘记减法公式

---

👉 **时间复杂度分析**

**滑动窗口为什么是 O(n)？**关键原因：

* 每个元素只会被 left 指针访问一次
* 每个元素只会被 right 指针访问一次

所以：

```
left 总移动次数 ≤ n
right 总移动次数 ≤ n
```

整体 O(n)
