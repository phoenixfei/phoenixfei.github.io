在数组和字符串题目中，如果你还在写双重循环。那说明你可能忽略了一个重要技巧：双指针。
双指针的本质是：
> **用两个指针在序列中协同移动，降低枚举复杂度。**

---

## 基本概念

双指针指的是：

* 使用两个索引变量
* 在遍历过程中动态移动
* 达成某种目标（查找 / 统计 / 优化）

基本形式：

```python
i, j = 0, 0
while j < n:
    ...
```

它的核心目标是：

> 把 O(n²) 的枚举优化到 O(n)

---

## 双指针的四大类型

### 对撞指针（左右夹逼）

典型结构：

```python
left, right = 0, n - 1
while left < right:
    if 条件成立:
        left += 1
    else:
        right -= 1
```

适用场景：

* 数组已排序
* 需要从两端逼近

经典题：

* 两数之和 II（有序数组）
* 三数之和
* 盛最多水的容器
* 回文判断

核心思想：

> 利用“有序性”缩小搜索空间。

---

### 快慢指针

一个指针走得快，一个走得慢。

```python
slow = 0
for fast in range(n):
    if 条件:
        nums[slow] = nums[fast]
        slow += 1
```

适用场景：

* 原地删除
* 去重
* 链表找环

经典题：

* 删除有序数组重复项
* 移动零
* 判断链表是否有环

核心思想：

> 用一个指针维护“有效区间”。

---

### 滑动窗口（双指针的特殊形式）

结构：

```python
left = 0
for right in range(n):
    加入元素
    while 不合法:
        left += 1
```

它是双指针的一种高级应用。

区别在于：

| 双指针       | 滑动窗口     |
| --------- | -------- |
| 不一定维护合法区间 | 始终维护合法区间 |

---

### 分离型双指针（双数组遍历）

两个指针分别遍历两个数组：

```python
i = j = 0
while i < m and j < n:
    if A[i] < B[j]:
        i += 1
    else:
        j += 1
```

典型应用：

* 合并两个有序数组
* 求交集
* 区间合并

## 例题解析

### [3. 无重复字符的最长子串](https://leetcode.cn/problems/longest-substring-without-repeating-characters/description/)

同向双指针

* 两个指针都从左向右（或同一方向）扫描
* 常用于 **滑动窗口**、**子序列/子数组问题**

```text
给定一个字符串 s ，请你找出其中不含有重复字符的 最长 子串 的长度。

示例 1:
输入: s = "abcabcbb"
输出: 3 
解释: 因为无重复字符的最长子串是 "abc"，所以其长度为 3。注意 "bca" 和 "cab" 也是正确答案。
```

```python
class Solution:
    def lengthOfLongestSubstring(self, s: str) -> int:
        ans = left = 0
        window = set()  # 维护从下标 left 到下标 right 的字符
        for right, c in enumerate(s):
            # 如果窗口内已经包含 c，那么再加入一个 c 会导致窗口内有重复元素
            # 所以要在加入 c 之前，先移出窗口内的 c
            while c in window:  # 窗口内有 c
                window.remove(s[left])
                left += 1  # 缩小窗口
            window.add(c)  # 加入 c
            ans = max(ans, right - left + 1)  # 更新窗口长度最大值
        return ans
```

* `left`、`right` 两个指针 **同向移动**
* 时间复杂度 O(n)

### [167. 两数之和 II - 输入有序数组](https://leetcode.cn/problems/two-sum-ii-input-array-is-sorted/description/)

相向双指针

* 一个指针从左，一个从右
* 常用于 **排序数组的两数和 / 三数和**、**反转数组**

```text
给你一个下标从 1 开始的整数数组 numbers ，该数组已按 非递减顺序排列  ，请你从数组中找出满足相加之和等于目标数 target 的两个数。
如果设这两个数分别是 numbers[index1] 和 numbers[index2] ，则 1 <= index1 < index2 <= numbers.length 。
 
示例 1：
输入：numbers = [2,7,11,15], target = 9
输出：[1,2]
解释：2 与 7 之和等于目标数 9 。因此 index1 = 1, index2 = 2 。返回 [1, 2] 。
```

```python
class Solution:
    def twoSum(self, numbers: List[int], target: int) -> List[int]:
        left = 0
        right = len(numbers) - 1
        while True:
            s = numbers[left] + numbers[right]
            if s == target:
                return [left + 1, right + 1]  # 题目要求下标从 1 开始
            if s > target:
                right -= 1
            else:
                left += 1
```

* 利用数组有序性快速逼近目标
* 时间复杂度 O(n)，比暴力 O(n²) 高效

### [15. 三数之和](https://leetcode.cn/problems/3sum/description/)

**三指针（Three Pointers）**：在一个序列（数组/链表）上同时维护三个指针，通过它们之间的相对位置控制逻辑或状态。

> 核心思想：多指针都是 **利用指针的相对位置维护状态**，用来减少嵌套循环、实现线性或接近线性扫描。

```text
给定一个包含 n 个整数的数组 nums，判断 nums 中是否存在三个元素 a ，b ，c ，使得 a + b + c = 0 ？
请找出所有和为 0 且 不重复 的三元组。

示例 1：
输入：nums = [-1,0,1,2,-1,-4]
输出：[[-1,-1,2],[-1,0,1]]
```

思路：

1. 固定第一个指针 `i`
2. 其余两个指针 `j`, `k` 在剩余数组上相向逼近，寻找符合条件的三元组
3. 三个指针都不回退，整体复杂度 O(n²)

```python
class Solution:
    def threeSum(self, nums: List[int]) -> List[List[int]]:
        nums.sort()
        ans = []
        n = len(nums)
        for i in range(n - 2):
            x = nums[i]
            if i > 0 and x == nums[i - 1]:  # 跳过重复数字
                continue
            if x + nums[i + 1] + nums[i + 2] > 0:  # 优化一
                break
            if x + nums[-2] + nums[-1] < 0:  # 优化二
                continue
            j = i + 1
            k = n - 1
            while j < k:
                s = x + nums[j] + nums[k]
                if s > 0:
                    k -= 1
                elif s < 0:
                    j += 1
                else:  # 三数之和为 0
                    ans.append([x, nums[j], nums[k]])
                    j += 1
                    while j < k and nums[j] == nums[j - 1]:  # 跳过重复数字
                        j += 1
                    k -= 1
                    while k > j and nums[k] == nums[k + 1]:  # 跳过重复数字
                        k -= 1
        return ans
```

* `i` 固定，`j`、`k` 左右逼近 → 三指针
* 原本三重循环 O(n³) → 优化为 O(n²)

### [18. 四数之和](https://leetcode.cn/problems/4sum/)

**四指针的典型模式：四重循环优化**

思路：

1. 固定前两个指针 `i`, `j`
2. 其余两个指针 `l`, `r` 在剩余数组上相向逼近
3. 四个指针共同确定满足条件的四元组

* 原本四重循环 O(n⁴) → 优化为 O(n³)
* 三指针 → 两重循环 + 双指针
* 四指针 → 两固定 + 双指针逼近

---

**多指针设计思路**

1. **明确每个指针的角色**

   * 固定指针 / 左右逼近指针 / 条件指针
2. **确定移动策略**

   * 根据条件判断哪个指针先移动
   * 保证不回退、不漏元素
3. **利用指针关系维护状态**

   * 比如窗口大小、和、最大/最小值、满足条件的组合数

## 其他知识点

👉 **双指针为什么能降复杂度？**

暴力枚举是：

```
每个 i 对应所有 j
```

双指针做的是：

> 利用某种单调性，让指针“只前进，不回头”。

因此：

```
每个元素最多访问两次
```

时间复杂度：

```
O(n)
```

---

👉 **双指针的核心前提**

双指针通常依赖以下性质之一：

1. 有序性
2. 单调性
3. 可维护区间合法性
4. 单调扩展或收缩

如果没有这些结构特征，一般不能用双指针。

---

👉 **常见错误**

* ❌ 忘记排序
* ❌ 指针移动方向写反
* ❌ 漏掉边界条件
* ❌ 死循环（指针不动）

---

👉 **什么时候想到双指针？**

看到这些信号：

* 有序数组
* 去重
* 区间
* 连续子数组
* 两端逼近
* 原地修改

优先考虑双指针。
